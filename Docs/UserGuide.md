# PushBoxz 使用说明

本文档说明 PushBoxz 项目中关卡编辑器、关卡列表、游戏内流程和创造模式的使用方式。

小猫往往讨厌黄瓜，在这里小猫会想办法用箱子把黄瓜挡住。PushBoxz 在经典的推箱子游戏的基础上，着重开发游戏内外对关卡编辑的工具，帮助游戏开发者/游戏玩家创建关卡。

Unity 版本：`2022.3.51f1c1`

以下内容，1-3为开发者视角，4-6为玩家视角，两个视角有独立但相似的关卡编辑器。

## 1. 工具入口

在 Unity 顶部菜单中打开：

- `Tools/PushBoxz/Level Editor`
- `Tools/PushBoxz/Level Registry`
- `Tools/PushBoxz/Rebuild Level Registry`

额外快捷入口：

- 在 Project 窗口中选中一个 `LevelDataAsset`，可在 Inspector 中点击打开关卡编辑器。
- 在 Project 窗口中右键一个 `LevelDataAsset`，可通过 PushBoxz 的右键菜单打开关卡编辑器。

## 2. 关卡编辑器

`Level Editor` 用于创建和编辑官方关卡资源。

### 2.1 关卡信息

顶部区域包含：

- `LevelDataAsset`：当前正在编辑的关卡资产。
- `New`：新建一个关卡资产并加载到编辑器。
- `Level ID`：关卡稳定 ID，用于运行时显示、存档和关卡列表识别。
- `Width` / `Height`：地图宽高，范围为 `2-16`。

新建官方关卡时，编辑器会根据当前官方关卡列表数量自动命名，例如 `L001`、`L002`。保存关卡时，资源文件名会自动改为与 `Level ID` 相同。

### 2.2 画笔

画笔列用于编辑网格：

- `Wall`：墙体，阻挡玩家和箱子。
- `Goal`：目的地，箱子需要被推到这里。
- `Box`：箱子初始位置。
- `Player`：玩家初始位置。
- `Erase`：擦除物体或目的地，并保留为地板格子。

关卡需要满足箱子数量等于目的地数量。

### 2.3 正推与倒拉设计模式

编辑器支持两种设计测试模式：

- `Push`：按 `F` 推动玩家正前方的箱子。
- `Pull`：按 `F` 将玩家正前方的箱子向玩家身后倒拉一格。

两种模式下：

- `WASD` 控制编辑器内的玩家移动。
- 玩家当前朝向会以箭头显示。
- 玩家和箱子都会受到墙体、箱子和地图边界阻挡。

倒拉模式适合从“接近完成”或“已经完成”的状态反向设计推箱子关卡。

### 2.4 自动生成

自动生成面板通过倒拉思路生成关卡，用于为开发者关卡设计提供思路（算法本身的设计能力，还不足以直接生成足够令人满意的关卡！）。

主要参数：

- `Boxes / Goals`：箱子和目的地数量，范围为 `1-8`。
- `Random Seed`：随机种子，用于复现生成结果。
- `Initial Wall Density`：初始墙密度，范围为 `0-1`。
- `Extra Wall Density`：补充墙密度，范围为 `0-1`。
- `Continue Push`：倾向于继续操作当前箱子的权重。
- `Change Direction`：倾向于绕到当前箱子其他方向再操作的权重。
- `Change Box`：倾向于切换到其他箱子的权重。

三个行为权重会相互制约，总和为 `1`。

自动生成的大致流程：

1. 先把箱子放在目的地上，构造一个已完成状态。
2. 随机生成一部分初始墙体。
3. 选择某个箱子旁边可站立的位置作为玩家起点。
4. 根据行为权重选择继续操作、换方向或换箱子。
5. 每次换方向或换箱子后，必须保证至少能真实倒拉一格。
6. 通过访问记录、箱子移动历史、反向回退检测和“箱子回到目标点后降低交互倾向”等规则减少无意义循环。
7. 如果生成过程进入局部死胡同，会尝试短窗口回退，而不是直接放弃整张地图。
8. 生成过程会记录倒推动作，用于还原一条可解路径。

### 2.5 关卡场景资产

`Level Scene Assets` 区域用于配置运行时共享表现资源：

- 可选预制体：地板、墙体、目的地、箱子、玩家。
- 音频：移动音效、推箱子音效、箱子到达目标点音效、通关音效、UI 按钮音效。
- UI：全局 TMP 字体。

这些配置会保存到 `LevelSceneBuilderRegistry` 中，因此关闭关卡编辑器再重新打开时，资源引用仍会保留。运行时生成关卡也会读取这些配置。

### 2.6 底部按钮

- `Save Level`：校验并保存当前关卡。
- `Open Level Registry`：打开官方关卡列表窗口。
- `Add To Level Registry`：把当前关卡加入官方关卡列表。

## 3. 关卡列表

打开路径：
Tools/PushBoxz/Level Registry

关卡列表决定哪些官方关卡会出现在游戏内关卡选择界面。

每一行包含：

- 关卡资产引用。
- `Enabled`：是否启用，关闭后不会出现在官方关卡选择中。
- `Unlocked`：是否初始解锁。

列表左侧可以拖拽调整顺序。运行时的关卡推进也会按照这个顺序进行：通关当前关卡后，下一个启用关卡会被解锁。

常用操作：

- `Add`：新增一条空关卡记录。
- 全部启用或全部禁用。
- 全部解锁或全部锁定。
- 删除某条列表记录。

如果需要重新扫描有效关卡资产，可以使用：
Tools/PushBoxz/Rebuild Level Registry

重建时会尽量保留已有的手动排序、启用状态和解锁状态，然后把新发现的有效关卡追加到列表末尾。

## 4. 游戏内流程

游戏内使用 Canvas UI 组织完整体验流程。

### 4.1 封面 UI

封面包含：

- `Start`：进入关卡选择。
- `Create`：进入游戏内自定义关卡编辑器。
- `Quit`：退出游戏。

### 4.2 关卡选择 UI

关卡选择界面支持分页显示。

官方模式：

- 显示 `LevelSceneBuilderRegistry` 中启用的官方关卡。
- 未解锁的关卡不可点击。
- 通关一个关卡后会解锁下一个关卡。
- 已通关关卡会通过存档标记。

创造模式：

- 显示玩家在游戏内创建的自定义关卡。
- 自定义关卡与官方 Unity 关卡资产分开存储。
- 关卡选择界面中有按钮可以在官方模式和创造模式之间切换。
- 创造模式中可以粘贴他人分享的关卡码，并点击 `Load` 导入关卡。

创造模式删除：

- 点击 `Delete` 后进入删除选择模式。
- 按钮文本会变为 `Select`。
- 此时点击某个自定义关卡会删除该关卡。
- 再次点击 `Select` 会退出删除模式，按钮文本恢复为 `Delete`。

创造模式导入关卡码：

- 在关卡码输入框中粘贴 `PBZ1-XXXX-XXXX` 形式的关卡码。
- 点击 `Load`。
- 如果校验通过，该关卡会保存为创造模式关卡，并刷新到关卡列表中。
- 如果关卡码不完整、被改错或内容不合法，会显示错误提示。
- 导入的关卡会使用创造模式自动 ID，例如 `C001`、`C002`。

### 4.3 游戏 HUD

进入关卡后显示：

- 当前关卡 ID 或名称。
- 当前步数。
- `Undo`：回撤一步。
- `Restart`：重置当前关卡。
- `Menu`：返回关卡选择。
- `Share`：复制当前关卡的关卡码到系统剪贴板。

基础操作：

- `WASD`：移动玩家。
- 格子步进模式下，朝箱子继续移动即可推动箱子。
- 连续移动模式下，可根据当前配置使用交互键推动箱子。

分享关卡码：

- 进入一个关卡后点击 `Share`。
- 游戏会把当前关卡编码成 `PBZ1-XXXX-XXXX` 形式的文本并复制到剪贴板。
- 该关卡码只包含玩法数据，不包含预制体、音效、UI 资源、通关状态或解锁状态。
- 其他玩家可以在创造模式关卡列表中粘贴并 `Load`，不需要联网。

### 4.4 通关 UI

当所有箱子都与目的地重合时，显示通关弹窗：

- `Continue`：返回关卡选择。
- `Next`：如果存在下一关，则直接进入下一关。

通关时会立刻解锁下一关，因此点击 `Next` 前下一关已经处于可进入状态。

## 5. 游戏内自定义关卡编辑器

游戏内自定义编辑器用于让玩家创建简单的创造模式关卡，不需要打开 Unity Editor 工具。

主要控件：

- 尺寸 Slider：设置正方形地图尺寸。
- 画笔按钮：墙、目的地、箱子、玩家、擦除。
- 模式按钮：
  - 初始显示 `Mode`。
  - 切换后显示当前模式 `Push` 或 `Pull`。
- 网格按钮：点击格子进行绘制，也可以按住鼠标左键扫过多个格子连续绘制。
- `Save`：校验并保存到创造模式关卡列表。
- `Back`：返回封面。

关卡 ID：

- 玩家不填写关卡 ID 时，保存后会自动生成短 ID，例如 `C001`。
- 如果关卡 ID 输入框中填写的是普通文本，会作为该自定义关卡的 ID 使用。
- 如果关卡 ID 输入框中粘贴的是 `PBZ1-XXXX-XXXX` 形式的关卡码，点击 `Save` 时不会立刻保存，而是会先把该关卡码加载到编辑器 Grid 中。
- 通过关卡码加载到 UI5 后，尺寸 Slider 会更新为该关卡宽高中的较大值，Grid 会刷新为关卡码中的墙、目的地、箱子和玩家。
- 玩家可以继续修改这个关卡，再次点击 `Save` 才会保存到创造模式关卡列表。

键盘测试：

- `WASD`：移动已放置的玩家。
- `F`：根据当前模式推动或倒拉玩家正前方的箱子。

保存自定义关卡前需要通过校验：

- 必须有且只有一个玩家。
- 至少有一个箱子和一个目的地。
- 箱子数量必须等于目的地数量。
- 玩家和箱子必须位于合法地板格子上。

保存成功后，自定义关卡会出现在关卡选择界面的创造模式列表中。

## 6. 关卡码说明

关卡码用于离线分享玩家自定义关卡。它不需要服务器，也不需要网络连接。

关卡码格式示例：

```text
PBZ1-XXXX-XXXX-XXXX
```

格式规则：

- `PBZ1` 表示 PushBoxz 第 1 版关卡码格式。
- 后续内容只使用大写字母、数字和连字符，方便复制、粘贴和转发。
- 关卡码内置校验码，如果输入错误会导入失败。

关卡码包含：

- 地图尺寸。
- 地形格子。
- 目的地。
- 箱子位置。
- 玩家位置。

关卡码不包含：

- 预制体、材质、音效、字体等本地资源。
- 玩家通关状态。
- 关卡解锁状态。
- 当前游玩过程中的撤销历史或临时状态。
